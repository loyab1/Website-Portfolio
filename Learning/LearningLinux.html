Learning Linux

Permissions

Linux systems support multiple users each with their own home directory, files, and permissions.

Permissions control what a user can do with the system its files and its directories. Permissions are represented by a 10 character string. Example: drwxrwxrwx

  • the first character represents whether its a file or directory
    • - Regular File
    • d Directory
  • The next 3 sets of characters rwx represents permissions for the owner, group, and others
    • rwx: all permissions
    • rw-: Read and write but not execute
    • r-x: Read and execute but not write.
  • The owner is whoever created the file or directory
  • The group is the group the file or directoy and user is assigned to
  • Other is considered everyone else

The chmod is how you change the permissions of a file or directory. it takes in 3 arguements u= (user), g= (group), and o= (other).
Example: chmod -R u=rwx,g=,o= DIRECTORY

The chown is a command that allows you to change the owner of a file or directory

Shell

Commands

  • $ - calls a variable that is created.
  • env - list all of the environment variables created.
  • export - Set a variable for the current shell session. Programs and scrips can also use this variable
  • unset - remove an environment variable from the current shell session.
  • head - display first few lines of text in a file. Default is 10 lines, can use the flag -n to specify
  • tail - display last few lines of text in a file. Default is 10 lines, can use the flag -n to specify
  • touch - Creates a file if it does not exist or updates access and modification time stamps
  • mv - Can rename a file or moves a file or directory from one location to another.
  • rm - removes a directory. the flag -r delets a directory and all its contents.
  • cp - Copy a file from one source to another. the -R flag copies all content
  • grep - Used to search for text in files. The -r flag can be used to search recursively through a directory
  • find - tool used for finding files and directories by name. Uses -name flag
  • wc - word count command to count the number of lines words and bytes in a file.
  • ln - create a link. to create a symbolic link use -s flag. target_path then link_path
  • top - view what resources are being used by system programs. Press o while running to sort
  • ps aux - list processes running on the machine
  • whoami - prints user currently logged in

Flags

Options that can be passed to a command to change how it behaves.

  • --help - use this flag after a command to print helpful information
  • > "filename.extension" - redirect stdout to a file
  • 2> "filename.extension" - redirect stderr to a file
  • < "filename.extension" - Redirects a file into a programs stdin

Writing Scripts

A script itself is a program. There are two types of programs compiled and interperted. A compiled program does not require a separate interperter to run like python, java, etc. It is pure machine code instruction.

Shebang - a special line at the top of the script that tells the shell which program to use to execute the file. for example: #! /usr/bin/env bash will use the bash program to interpert the script.

Multiple commands can be run on a single line by separating with a ;
If you only want the second command to run when the first succeeds use &&

If a script does not want to run, make sure it has an executable permission. chmod +x

Concepts

  • Piping

    • Piping is taking the output (stdout) of one program and inputting (stdin) it into another program. Operator is |

  • stdin, stdout, stderr

Command Tips

touch can be useful when creating scripts because it will check if the file exist without changing any or creates a new file if necessary

The PATH variable is a list of directories that the shell will look into when running a command. To add a directory to the PATH without overwriting, make sure to use the export command and reference the existing variable.

Example: export PATH="$PATH:/new/directory"

Creating your Own Shell Experience

Bash and Zsh have configuration files that run each time a new session is started. They can be used to setup aliases, functions, and environment variables. These files are hidden inside the home directory. Bash file is .bashrc while Zsh is .zshrc if the files arent there you can create them.

Reading the Manual

to access the manual you will use man command. The command will work for many of the built in commands.

  • to search the manual you will press the / key
    • to jump to the next result press n key
    • to go back press N key
  • to page through the manual press space bar
  • to quit the manual press q

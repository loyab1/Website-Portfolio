CSS: Styling the Experience

CSS (Cascading Style Sheets) is the paint, interior design, and architecture that makes our raw HTML look beautiful and function responsively on any screen size.

What is CSS?

If HTML is the skeleton, CSS is the skin and clothing. CSS uses "Selectors" to target specific HTML tags, classes, or IDs, and applies visual rules to them.

.card {
  background-color: white;
  border-radius: 12px;
}

In the example above, the dot (.) means we are targeting a "Class". Any HTML element with class="card" will instantly receive a white background and softly rounded corners.

CSS Variables (Custom Properties)

At the very top of our `style.css` file, you might notice variables that look like this: --bg-color: #f8fafc;. CSS Variables allow us to define our color palette exactly once. If we ever want to build a "Dark Mode", we don't have to change the color on 100 different lines of code; we simply change the variable at the top, and the entire website updates instantly!

Flexbox and CSS Grid

Centering items on a webpage used to be notoriously difficult. We solved this by using two modern CSS layout modules:

  • Flexbox (display: flex;): Used to align items in a single row or column. We used Flexbox to perfectly space out the links in our top Navigation Bar.
  • CSS Grid (display: grid;): Used for complex two-dimensional layouts. We used Grid on the Circuit Builder to create the beautiful dotted workspace background, and on the Boolean Logic page to organize the Truth Tables neatly.

Mobile Responsiveness (Media Queries)

A website needs to look good on a 27-inch monitor AND a 5-inch smartphone. To achieve this, we used Media Queries.

@media screen and (max-width: 768px) {
  /* Mobile styles go here */
}

This code tells the browser: "If the screen width is 768 pixels or smaller, apply these new rules." We used this to automatically stack our side-by-side flex layouts into vertical columns so that users don't have to zoom in on their phones!

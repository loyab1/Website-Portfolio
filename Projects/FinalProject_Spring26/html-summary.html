HTML: The Foundation of the Web

HTML (HyperText Markup Language) provides the raw structure and skeleton for our entire website.

What is HTML?

Think of HTML as the framing of a house. Before you can paint the walls (CSS) or wire the electricity (JavaScript), you need to build the rooms. HTML uses a system of Tags to define what kind of content is on the screen.

<h1>This is a Header</h1>
<p>This is a paragraph of text.</p>

A tag is enclosed in angle brackets. Most elements have an opening tag <p> and a closing tag </p> that wrap around the content.

Attributes: Classes and IDs

Tags alone aren't enough. We need to identify specific pieces of our website so we can style them later or interact with them using code. We do this using Attributes.

  • ID (id="calculator"): An ID is unique. It is like a social security number for an HTML element. We used IDs heavily in our Circuit Builder so the JavaScript engine knows exactly which logic gate to connect a wire to.
  • Class (class="card"): A Class is reusable. We used the "card" class throughout this website so that the CSS rules can instantly apply a white background, rounded corners, and a shadow to multiple different boxes at once!

Semantic HTML in Our Project

Instead of making everything a generic box (a <div>), modern HTML uses "Semantic Tags". These are tags that actually describe their meaning to the browser and to screen-readers for accessibility.

  • <nav>: We used this at the top of every page to wrap our navigation links.
  • <main>: We used this to wrap the core content of the page, separating it from the headers and footers.
  • <section> and <article>: We used these to logically group the algorithm descriptions and the logic gate truth tables.

By using semantic HTML, our code is cleaner, easier to read, and better for Search Engine Optimization (SEO).

Native Dropdowns and Future-Proofing

In basic HTML, you can create fully functional dropdown menus without writing a single line of CSS or JavaScript. There are two primary native tags for this:

  • The Form Dropdown (<select>): Used when you want a user to pick an option (like choosing a language or country). It works completely out of the box and natively adapts to mobile device UI (like the iOS scrolling wheel).
  • The Expandable Menu (<details> and <summary>): Used to create a native "click to expand" widget, perfect for FAQs or simple navigation menus. Clicking the summary text drops down the nested content.

Are these future-proof for CSS and JS? Yes! In fact, native HTML elements are the most future-proof choice you can make. Because they are built directly into web browsers, they are guaranteed to be fully accessible for screen-readers and perform perfectly on any new device. Once you establish this strong, semantic HTML foundation, you can safely layer CSS on top to make them beautiful, and attach JavaScript to add complex behaviors, without worrying about the core functionality breaking.

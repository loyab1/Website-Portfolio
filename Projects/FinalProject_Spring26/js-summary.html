JavaScript: The Brains of the Operation

JavaScript (JS) is the programming language that brings our website to life. Without JS, the website would just be a static painting. JS allows us to calculate math, drag-and-drop elements, and animate sorting algorithms!

Variables: Storing Information

Think of a variable as a labeled box where we can store data to use later. We declare variables using words like let or const.

let isWiring = false;
const workspace = document.getElementById("workspace");

In our Circuit Builder, we created a variable called isWiring. We set it to false initially. When a user clicks on an output port, we change the variable to true. The code constantly checks this "box" to know whether or not it should draw an SVG wire following the user's mouse!

Conditionals (If/Else Statements)

Conditionals allow our code to make decisions. They are the crossroads of programming. "IF this condition is true, DO this. ELSE, do that."

if (node.type === 'AND') {
  return (a === 1 && b === 1) ? 1 : 0;
} else if (node.type === 'OR') {
  return (a === 1 || b === 1) ? 1 : 0;
}

We used massive conditional blocks (like switch statements) in our Circuit Builder's logic engine. By checking the string text of the node.type variable, the JavaScript engine knows exactly which mathematical Boolean logic to apply to the signals traveling through the wire.

The DOM (Document Object Model)

The DOM is how JavaScript talks to HTML. When the browser loads HTML, it converts all the tags into "Objects" that JavaScript can read and modify.

By using commands like document.getElementById(), we can grab an HTML element (like our lightbulb image). We can then use JS to change its CSS class, literally making the bulb light up yellow when an algorithm finishes or an AND gate receives two 1 signals!

Arrays and State Machines

An Array is a list of data, like [5, 2, 8, 1, 9]. On our Algorithm Visualizer page, we store the heights of all the visual bars inside an Array.

To animate the sorting, we used an advanced JS concept called a Generator State Machine. Instead of sorting the array instantly, the Generator "pauses" the code after every single swap. This allowed us to build Play, Pause, and Step buttons, slowly revealing the algorithm's thought process step-by-step!
